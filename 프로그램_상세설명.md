# GHJ Codex V.03 프로그램 상세 설명

## 1. 프로그램 목적

`ghj_codex_V_03.py`는 기존 Jupyter Notebook(`DS_GHJ_VERSION_04.ipynb`)에서 Selenium으로 수행하던 KRX 수급 데이터 조회 작업을 Flask 웹 프로그램으로 바꾼 도구입니다.

기존 노트북은 KRX 정보데이터시스템에 접속해 화면을 클릭하고 엑셀을 내려받은 뒤, 여러 기간의 데이터를 합쳐 분석했습니다. 이 프로그램은 같은 목적을 더 쉽게 실행할 수 있도록 웹 화면에서 아이디, 비밀번호, OpenAPI 키만 입력하면 전체 수집과 분석이 한 번에 진행되도록 구성했습니다.

## 2. 실행 방법

PowerShell에서 프로그램 폴더로 이동합니다.

```powershell
cd "C:\Users\MK-Desktop\Documents\외국인,기관수급현황실시간대시보드 제작하기"
```

프로그램을 실행합니다.

```powershell
python ghj_codex_V_03.py
```

브라우저가 자동으로 열립니다. 열리지 않으면 아래 주소로 접속합니다.

```text
http://127.0.0.1:5000
```

## 3. 입력값 설명

### KRX 아이디

KRX 정보데이터시스템 계정 아이디입니다.

회원가입 주소:

```text
https://data.krx.co.kr/
```

### KRX 비밀번호

KRX 정보데이터시스템 로그인 비밀번호입니다. 프로그램은 조회 실행 중에만 사용하며, 파일에 저장하지 않습니다.

### OpenAPI 키

KRX OpenAPI 사이트에서 발급받는 API 인증키입니다.

발급 사이트:

```text
https://openapi.krx.co.kr/
```

OpenAPI 사이트에 로그인한 뒤 `인증키`, `API Key`, `활용신청`, `OpenAPI 신청` 관련 메뉴에서 키를 발급받습니다.

### 기준일

분석 기준 날짜입니다. 기본값은 오늘 날짜입니다.

프로그램은 이 기준일을 기준으로 아래 데이터를 자동 조회합니다. 사용자가 기간을 선택할 필요는 없습니다.

- 기준일 기준 6개월 누적 데이터
- 기준일 기준 3개월 누적 데이터
- 기준일 기준 1개월 누적 데이터
- 기준일 이전 최근 거래일 5개 일별 데이터

### 시장

조회할 시장 범위입니다.

- `ALL`: 전체
- `KOSPI`: 유가증권시장
- `KOSDAQ`: 코스닥
- `KONEX`: 코넥스

## 4. 자동 분석 기간

분석 기간은 혼란을 줄이기 위해 화면에서 선택하지 않도록 고정했습니다.

프로그램은 항상 아래 범위를 한 번에 조회합니다.

- 6개월 누적
- 3개월 누적
- 1개월 누적
- 최근 거래일 5개 일별 데이터

주말과 휴장일은 프로그램이 자동으로 건너뜁니다.

내부 기준값은 아래와 같습니다.

- 최근 거래일 분석 개수: 5개
- 휴장일 탐색 범위: 기준일 이전 최대 20일
- KRX 요청 대기 시간: 요청 1건당 최대 30초

## 5. 기존 Notebook 기능과의 대응

기존 노트북에서 수행하던 핵심 작업은 아래와 같습니다.

1. KRX 로그인
2. 투자자별 순매수 상위종목조회 화면 이동
3. 기관합계 6개월 데이터 다운로드
4. 외국인 6개월 데이터 다운로드
5. 기관합계 3개월 데이터 다운로드
6. 외국인 3개월 데이터 다운로드
7. 기관합계 1개월 데이터 다운로드
8. 외국인 1개월 데이터 다운로드
9. 최근 거래일 5일치 기관합계 일별 데이터 다운로드
10. 최근 거래일 5일치 외국인 일별 데이터 다운로드
9. 모든 데이터를 `base_df`로 통합
10. `last_df` 피벗 생성
11. 엑셀 저장
12. 기본 그래프 생성

`ghj_codex_V_03.py`는 이 흐름을 웹 버튼 한 번으로 실행합니다.

## 6. 데이터 수집 방식

기존 노트북은 Selenium으로 브라우저를 열고, 사용자가 클릭하듯 KRX 화면을 조작했습니다.

현재 프로그램은 Selenium 대신 KRX 내부 JSON 요청을 직접 호출합니다.

주요 장점:

- 크롬 드라이버 설치가 필요 없습니다.
- 다운로드 폴더에서 엑셀 파일을 찾는 과정이 필요 없습니다.
- 수집된 데이터를 바로 DataFrame으로 변환합니다.
- 통합 엑셀과 시각화를 자동 생성합니다.

## 7. 결과 엑셀 설명

조회가 완료되면 아래 위치에 엑셀 파일이 저장됩니다.

```text
outputs\YYYYMMDD\YYYY-MM-DD_통합파일.xlsx
```

웹 화면의 `엑셀 다운로드` 버튼으로도 받을 수 있습니다.

엑셀에는 두 개의 시트가 들어갑니다.

### base 시트

수집된 원천 데이터를 기간과 투자자 구분까지 포함해 모두 합친 시트입니다.

주요 컬럼:

- `종목코드`
- `종목명`
- `거래량_매도`
- `거래량_매수`
- `거래량_순매수`
- `거래대금_매도`
- `거래대금_매수`
- `거래대금_순매수`
- `period(D-00)_start`
- `period(D-00)_end`
- `buyer`

### last 시트

기존 노트북의 `last_df`에 해당합니다.

종목코드와 종목명을 기준으로 기간별, 투자자별 순매수 데이터를 피벗한 결과입니다.

예시 컬럼:

- `1개월누적_외국인`
- `1개월누적_기관합계`
- `3개월누적_외국인`
- `3개월누적_기관합계`
- `6개월누적_외국인`
- `6개월누적_기관합계`
- `1일전_외국인`
- `1일전_기관합계`

1개월보다 짧은 구간은 누적 컬럼이 아니라 `1일전`, `2일전`처럼 개별 거래일 단위 컬럼으로 표시됩니다.

## 8. 화면 시각화 설명

조회가 완료되면 웹 화면에서 기본 시각화를 바로 확인할 수 있습니다.

### 1개월누적 외국인 순매수 TOP10

1개월 누적 기준 외국인 순매수가 가장 큰 상위 10개 종목을 보여줍니다. 같은 종목에 대한 기관합계 값도 같이 비교합니다.

### 외국인 vs 기관합계 산점도

각 종목의 1개월 누적 외국인 순매수와 기관합계 순매수를 비교합니다.

오른쪽 위에 가까울수록 외국인과 기관이 모두 순매수한 종목입니다.

### 최근 거래일 평균 순매수

최근 거래일별 외국인과 기관합계의 평균 순매수 흐름을 보여줍니다.

### 1개월 / 3개월 / 6개월 누적 평균 순매수

단기, 중기 관점에서 외국인과 기관합계의 평균 순매수 규모를 비교합니다.

### 주요 종목 미리보기 테이블

1개월 누적 외국인 순매수 기준 상위 종목을 표로 보여줍니다.

## 9. 로딩 진행률

조회 중에는 로딩 화면과 퍼센트가 표시됩니다.

실제 작업은 아래 순서로 진행됩니다.

1. KRX 로그인
2. 6개월 데이터 수집
3. 3개월 데이터 수집
4. 1개월 데이터 수집
5. 최근 거래일 자동 탐색
6. 최근 거래일 데이터 수집
7. 피벗 생성
8. 시각화 생성
9. 엑셀 저장

웹 요청 방식의 특성상 퍼센트는 정확한 서버 내부 단계와 1:1로 연결된 값이 아니라, 사용자가 작업 진행 중임을 알 수 있도록 표시하는 진행 안내입니다.

## 10. 오류 대응

### KRX 로그인 실패

아이디나 비밀번호를 다시 확인합니다.

### OpenAPI 키를 입력해야 합니다

OpenAPI 키 입력칸이 비어 있으면 실행되지 않습니다.

### 최근 거래일을 찾지 못했습니다

기준일 주변에 휴장일이 많거나 너무 과거/미래 날짜를 입력했을 수 있습니다.

해결 방법:

- 기준일을 실제 거래일 근처로 바꿉니다.
- 기준일을 실제 거래일에 가까운 날짜로 바꿉니다.

### KRX API 오류 또는 세션 만료

KRX 사이트 상태, 계정 로그인 상태, 네트워크 상태를 확인합니다.

## 11. 보안 참고

프로그램은 입력된 KRX 비밀번호와 OpenAPI 키를 코드 파일에 저장하지 않습니다.

다만 웹 화면에 입력한 값은 실행 중인 로컬 프로세스에서 조회 요청을 보내기 위해 일시적으로 사용됩니다. 공용 PC에서는 실행 후 브라우저를 닫고 터미널도 종료하는 것을 권장합니다.
